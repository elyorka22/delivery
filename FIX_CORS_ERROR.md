# Исправление CORS ошибки

## Проблема
CORS блокирует запросы, потому что в Railway переменная `FRONTEND_URL` указывает на старый Vercel URL.

**Текущий Vercel URL:**
```
https://delivery-3ve4cvlbd-elyors-projects-9ae97d55.vercel.app
```

**Старый URL в Railway (неправильно):**
```
https://delivery-alpha-six.vercel.app
```

---

## Решение

### Обновить FRONTEND_URL в Railway

1. Зайдите на https://railway.app
2. Откройте ваш проект → сервис (backend)
3. Перейдите в **Settings** → **Variables**
4. Найдите переменную `FRONTEND_URL`
5. Нажмите на нее для редактирования
6. Измените значение на ваш текущий Vercel URL:
   ```
   https://delivery-3ve4cvlbd-elyors-projects-9ae97d55.vercel.app
   ```
7. Нажмите **Save**
8. Railway автоматически перезапустит сервер

---

## Как получить правильный Vercel URL

1. Зайдите на https://vercel.com
2. Откройте ваш проект
3. Посмотрите на URL вверху страницы или в **Domains**
4. Скопируйте полный URL с `https://`
5. Без слеша в конце

---

## Важно

- ✅ Используйте полный URL с `https://`
- ✅ Без слеша в конце
- ✅ Это должен быть ваш реальный Vercel URL (не preview URL)

---

## После исправления

1. Railway автоматически перезапустит сервер (1-2 минуты)
2. Обновите страницу в браузере (Ctrl+F5 или Cmd+Shift+R)
3. Попробуйте войти снова

---

## Проверка

После обновления `FRONTEND_URL` в Railway:

1. Проверьте логи в Railway:
   - **Deployments** → последний деплой → **Logs**
   - Должны увидеть: `Server is running on port 8080`

2. Попробуйте войти снова - CORS ошибка должна исчезнуть

---

## Если проблема осталась

1. Убедитесь, что `FRONTEND_URL` в Railway точно совпадает с вашим Vercel URL
2. Проверьте, что нет лишних пробелов или символов
3. Убедитесь, что Railway перезапустил сервер (проверьте логи)

---

## Готово! ✅

После обновления `FRONTEND_URL` в Railway CORS ошибка должна исчезнуть и вход должен работать.
