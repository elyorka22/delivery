# План миграции на Firebase

## Что нужно знать

Firebase использует **NoSQL базу данных (Firestore)**, а у нас сейчас **PostgreSQL (SQL)**. Это потребует изменений в структуре данных.

---

## Что нужно сделать

### 1. Создать проект Firebase
- Зарегистрироваться на https://firebase.google.com
- Создать новый проект
- Включить Firestore Database

### 2. Установить Firebase SDK
- Установить `firebase-admin` для backend
- Настроить подключение

### 3. Переписать схему данных
- Адаптировать SQL схему под NoSQL структуру
- Создать коллекции вместо таблиц

### 4. Переписать контроллеры
- Заменить Prisma запросы на Firebase запросы
- Адаптировать логику под NoSQL

---

## Важно

Это **большая работа** - нужно переписать:
- Все модели данных
- Все контроллеры
- Логику работы с базой данных

**Время:** ~2-3 часа работы

---

## Альтернатива

Можно использовать **Firebase Realtime Database** вместо Firestore, но это тоже NoSQL.
