# Шаги миграции на Firebase

## Что нужно сделать

### Шаг 1: Создать проект Firebase
1. Зайдите на https://firebase.google.com
2. Создайте аккаунт (если нет)
3. Создайте новый проект
4. Включите **Firestore Database** (не Realtime Database)
5. Выберите режим: **Production mode** (для начала)

### Шаг 2: Получить credentials
1. В Firebase Console → **Project Settings** → **Service accounts**
2. Нажмите **"Generate new private key"**
3. Скачайте JSON файл с credentials
4. Сохраните его (нужен будет для Railway)

### Шаг 3: Установить Firebase SDK
Я установлю `firebase-admin` в backend

### Шаг 4: Настроить подключение
Создам утилиту для работы с Firebase

### Шаг 5: Переписать контроллеры
Адаптирую все контроллеры под Firestore

---

## Важно знать

Firebase Firestore - это **NoSQL база данных**:
- Коллекции вместо таблиц
- Документы вместо строк
- Нет JOIN запросов (нужно делать несколько запросов)
- Другая структура данных

Это потребует переписать:
- Все модели данных
- Все контроллеры
- Логику работы с базой

**Время:** ~2-3 часа работы
